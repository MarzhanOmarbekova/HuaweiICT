# ⚡ VoltAI: Intelligent Wind Energy Optimization Platform

## 💡 О проекте

**VoltAI** — это инновационная платформа, которая использует комбинацию AI (на базе Huawei MindSpore), Blockchain (Huawei BCS) и Cloud-технологий для эффективного использования ветровой энергии.

**Основные цели:**
1. **AI Wind Optimization Engine:** Нахождение оптимальных точек для установки ветряных турбин на основе геоаналитики (ветер, рельеф, ограничения).
2. **P2P Energy Marketplace:** Децентрализованный и прозрачный обмен излишками энергии между пользователями через блокчейн.

На данный момент мы реализовали архитектуру бэкенда и ключевую API-точку с **заглушкой (Mock)**, имитирующей работу AI.

---

## ⚙️ Технологический стек

| Компонент | Технологии | Роль |
| :--- | :--- | :--- |
| **Backend** | **Django**, **Django REST Framework (DRF)**, **requests** | API-интерфейс, управление задачами и сохранение результатов. |
| **AI Layer (План)** | Huawei MindSpore / ModelArts | Обучение и развертывание моделей оптимизации и прогнозирования. |
| **Database** | PostgreSQL (сейчас SQLite) | Хранение задач, результатов AI и данных маркетплейса. |

## 🚀 Как запустить Backend (Для разработчиков)

### Требования
* Python 3.9+

### Пошаговая инструкция

1. **Перейдите в корневую папку проекта** (`Backend`).
2. **Активируйте виртуальное окружение** (`venv\Scripts\activate `).
3. **Установите все зависимости** (Django, DRF, requests):
    ```bash
    pip install -r requirements.txt
    ```
4. **Перейдите в папку** (`volt_ai_backend`).
5. **Подготовьте базу данных (создайте и примените миграции):**
    ```bash
    python manage.py makemigrations optimization
    python manage.py migrate
    ```
6. **Запустите сервер Django:**
    ```bash
    python manage.py runserver
    ```

Сервер будет доступен по адресу: **`http://127.0.0.1:8000/`**

## 📞 API Endpoint (AI Wind Optimization)

Ключевая точка входа для взаимодействия с AI-модулем. Логика **Mock** возвращает псевдослучайные, но валидные данные, чтобы фронтенд мог начать разработку карты и интерфейса.

### 1. Запуск оптимизации

| Метод | URL | Описание |
| :--- | :--- | :--- |
| `POST` | `/api/optimize-wind/` | Принимает область и количество турбин, запускает имитацию AI-анализа. |

### 2. Входящий JSON (Request Body)

| Поле | Тип | Описание | Обязательно |
| :--- | :--- | :--- | :--- |
| `area_polygon` | `Array<Object>` | Список координат `{lat: float, long: float}`, определяющих область. | Да |
| `num_stations` | `Integer` | Количество турбин для размещения (минимум 1). | Да |
| `save_results` | `Boolean` | Если `true`, результат сохраняется в базе данных. | Нет (Default: `false`) |
| `task_name` | `String` | Имя для задачи. | Нет |

**Пример вызова:**
Используйте этот JSON для тестирования через Postman или в коде фронтенда.

```json
{
  "area_polygon": [
    {"lat": 43.250, "long": 76.940},
    {"lat": 43.260, "long": 76.940},
    {"lat": 43.260, "long": 76.950},
    {"lat": 43.250, "long": 76.950}
  ],
  "num_stations": 4,
  "save_results": true,
  "task_name": "Test Run Almaty Area"
}
```
**Исходящий JSON (Response Body)**

При успешном выполнении возвращается статус 200 OK и результат:

```json
{
  "optimal_points": [
    {
      "lat": 43.25123,
      "long": 76.94198,
      "efficiency": 0.94  
    },
    // ... итого N точек ...
  ],
  "predicted_energy": {
    "1_month": 12500,  // Прогноз (kWh, Integer)
    "3_months": 37000,
    "6_months": 75000,
    "12_months": 150000
  }
}
```
